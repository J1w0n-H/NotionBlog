# NotionBlog 프로젝트 전체 설명서

> **대상:** 기술 배경 없는 경영진  
> **목적:** 이 사이트가 무엇인지, 어떻게 작동하는지, 어떤 역사를 거쳐 지금에 이르렀는지 한 문서로 이해

---

## 1. 한 줄 요약

**Jiwon Hwang의 개인 포트폴리오 + 블로그 웹사이트.** Notion(노션)을 글쓰기 도구로 쓰고, 그 내용을 자동으로 웹사이트에 게시하는 구조.

---

## 2. 이 사이트가 하는 일

방문자 입장에서 보면 세 가지 영역으로 구성됩니다.

| 영역 | 설명 | 접근 URL |
|------|------|----------|
| **피드(Feed)** | 블로그 글 목록 + 이력서 + 자기소개 | `j1w0n-log.vercel.app/` |
| **글 상세** | 개별 블로그 글 읽기 | `j1w0n-log.vercel.app/[글제목]` |
| **About** | 상세 자기소개 (경력 스토리 전문) | `j1w0n-log.vercel.app/about` |

---

## 5. 화면별 상세 설명

### 5-1. 메인 피드 (홈 화면)

**3열 레이아웃 (데스크톱 기준)**

```
┌──────────────┬─────────────────────────┬─────────────────┐
│   좌측 Nav    │       중앙 피드          │  우측 패널       │
│              │                         │  (글 열면 나옴)  │
│ · 섹션 이동  │  [프로필 카드]           │                 │
│ · 카테고리   │  $ whoami               │  Post Detail    │
│   필터        │  Jiwon Hwang            │  또는           │
│ · 태그 칩    │  Built it / Broke it /  │  About 상세     │
│ · 검색창     │  Mastered why.          │                 │
│              │                         │                 │
│ · Background │  [이력서 타임라인]       │                 │
│   섹션        │  (교육 + 경력)          │                 │
│              │                         │                 │
│              │  [블로그 글 카드들]      │                 │
│              │  카테고리별 그룹화       │                 │
└──────────────┴─────────────────────────┴─────────────────┘
```

**각 구성요소 설명:**

- **프로필 카드:** 이름, 태그라인 ("Built it. Broke it. Mastered why."), 주요 수치 (4 yrs / 200+ nodes / ISMS-P). "About →" 링크로 상세 자기소개 진입.
- **빌드 리본:** 상단 보라색 띠. "I designed & built this site — Next.js · Notion API · self-hosted · view source ↗" 표시. 홈 화면에서만 보임.
- **이력서 타임라인 (Background 섹션):** `site.config.js`에 설정된 교육/경력 데이터를 타임라인 형식으로 렌더링. 키워드 태그에 마우스 오버 시 상세 설명 툴팁 표시.
- **글 카드:** 카드에 썸네일 + 제목 + 날짜 표시. 마우스 오버 시 카드가 뒤집히며 요약문 표시 (summary 있는 글 한정). 카드 클릭 → 우측 패널에 글 열림 (페이지 이동 없음).

**모바일:** 1열로 전환, 좌측 Nav는 상단 고정 가로 스크롤 바로 변경.

### 5-2. 글 상세 (Post Detail)

**데스크톱:** 피드 오른쪽에 패널 형태로 슬라이드인. 피드 배열이 어두워지며 글에 집중.

```
┌──────────┬──────────┬──────────────────────────────────┐
│  Nav     │  피드    │  [Close >>]                      │
│ (축소됨) │ (어두움) │  ─────────────────────────────── │
│          │          │  글 제목                          │
│          │          │  날짜 · 카테고리 · 읽기 시간      │
│          │          │                                  │
│          │          │  본문 (Notion 렌더링)             │
│          │          │            │  [목차(TOC)]        │
│          │          │            │   H2 ─── pct%      │
│          │          │            │   H3 ──              │
│          │          │  [번역 버튼] KO/EN                │
│          │          │  [댓글 (Cusdis)]                 │
└──────────┴──────────┴──────────────────────────────────┘
```

**모바일:** 전체 화면으로 전환. 뒤로가기 버튼으로 피드 복귀.

**주요 기능:**
- **읽기 진행률 바:** 스크롤에 따라 상단에 진행률 표시 (목차 없는 글 한정)
- **목차 (TOC):** H2/H3 헤딩 자동 추출. 현재 읽는 위치 하이라이트. 클릭 시 해당 섹션으로 이동.
- **번역:** EN↔KO 버튼. 클릭 시 자체 번역 API를 통해 번역문을 원문 옆에 나란히 표시.
- **댓글:** Cusdis 위젯 (서드파티 댓글 서비스).
- **코드 하이라이팅:** Bash, Python, SQL, Go, Rust 등 25개 언어 지원. Mermaid 다이어그램도 지원.

### 5-4. 디자인 시스템

**컬러 팔레트 (다크 테마 기본):**

| 토큰 | 용도 | 느낌 |
|------|------|------|
| `--accent` | 보라 (#9b6cff) | 강조, 선택 상태 |

**카테고리 색상:** 8가지 색상 슬롯이 카테고리 순서에 따라 자동 배정. 별도 설정 불필요.

- 본문: Pretendard (한국어 최적화)
- 코드: JetBrains Mono

- 카드 호버 → 3px 위로 리프트 + 보라 글로우

- 태그라인 그라디언트 스크롤
- 터미널 커서 깜빡임
- `@media (prefers-reduced-motion)` 처리 포함 (모션 민감 사용자 배려)

## 6. 데이터 흐름 (글이 화면에 나타나는 과정)

```
① Notion에서 글 작성/수정
         │
         ▼
② 사이트 빌드 트리거 (Vercel, 자동 또는 수동)
         │
         ▼
③ Next.js가 Notion API 호출
   ├── @notionhq/client → 데이터베이스에서 글 목록(메타 정보) 가져오기
   └── notion-client → 각 글의 본문 블록 가져오기
         │
         ▼
④ 정적 페이지 생성 (SSG)
   모든 페이지를 미리 HTML로 만들어 저장
   → 빠른 로딩, SEO 우수
         │
         ▼
⑤ Vercel CDN에 배포
         │
         ▼
⑥ 방문자가 접속 → 미리 만들어진 HTML 즉시 전달
   (이후 React가 인터랙션 처리)
         │
         ▼
⑦ 글 클릭 → React Query가 해당 글 내용 API로 요청
   (이미 캐시된 경우 즉시 표시)
```

**재검증 (revalidate):** 30초마다 백그라운드에서 최신 데이터 확인. Notion에서 글을 수정하면 최대 30초 후 사이트에 반영.

**강제 갱신 API:** `/api/revalidate` 엔드포인트. 즉시 업데이트 필요 시 호출.

## 7. 주요 설정 포인트

모든 개인화 설정은 루트의 [`site.config.js`](site.config.js) 파일 하나에 집중되어 있습니다.

```
site.config.js가 담고 있는 것:
├── 프로필 (이름, 이메일, LinkedIn, GitHub, 역할, 소개)
├── 교육 이력 (학교명, 학위, 기간, 수강 과목, 연구실)
├── 경력 (회사명, 역할, 기간, 주요 성과)
├── 프로젝트 목록
├── 블로그 제목/설명/기본 테마(dark)
├── Notion 페이지 ID (어떤 데이터베이스와 연결할지)
├── Google Analytics ID
├── 검색 엔진 인증 코드 (Google, Naver)
├── 댓글 시스템 설정 (Cusdis / Utterances)
└── 번역 설정 (지원 언어, 기본 언어)
```

### Phase 3 — Notion API 붕괴 위기 + 번역 기능 (2025년 10월)
**커밋 수:** 55개 (하루 안에 집중)

Notion이 공식 API 버전을 `2025-09-03`으로 업그레이드하면서 기존 비공식 SDK가 작동을 멈춤. 사이트 빌드 자체가 실패하는 상황.

이 과정에서 AI(Claude)와 함께 단 하루 만에:
- 공식 Notion SDK (`@notionhq/client`)로 데이터베이스 조회 로직 전면 교체
- 썸네일 추출 로직 재구현
- 언어 감지 기능 추가
- 한국어↔영어 번역 시스템 구현 (Google/MyMemory/Lingva 폴백 체인)
- 번역 API 프록시 서버 구현

> 위기를 기회로 전환한 시점. 단순 마이그레이션을 넘어 새로운 기능을 추가.

## 9. 페이지별 URL 구조

|----------|------|
| `/` | 메인 피드 |

| `/api/translate` | 번역 프록시 |

## 10. Notion 데이터베이스 구조

Notion 데이터베이스의 각 행(Row)이 하나의 블로그 글이 됩니다.

| Notion 필드명 | 타입 | 역할 |
|--------------|------|------|
| `Title` | 제목 | 글 제목 |
| `Slug` | 텍스트 | URL 경로 (`/my-post`) |
| `Status` | 선택 | `Public` / `Private` / `PublicOnDetail` |
| `Type` | 선택 | `Post` / `Paper` / `Page` |
| `Category` | 선택/다중선택 | 카테고리 (색상 자동 배정) |
| `Tags` | 다중선택 | 태그 필터링용 |
| `Date` | 날짜 | 게시일 |
| `Summary` | 텍스트 | 카드 뒷면 요약문 |
| `Thumbnail` | 파일/커버 | 카드 썸네일 이미지 |
| `Lang` | 선택 | 언어 (`en` / `ko`) |

**공개 규칙:**
- `Status = Public` → 피드 + 상세 모두 표시
- `Status = PublicOnDetail` → 피드 목록에는 없고 URL 직접 접근 가능
- `Status = Private` → 완전 비공개 (빌드에 포함 안 됨)
- `Tags`에 `Pinned` 포함 → 피드 상단 고정 표시
